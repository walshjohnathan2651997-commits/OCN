## 结论先行

- **方法配套度较高：** 双分支模型、共享阈值序数校准、样本级升级损失、R(k) 映射、硬截断、置信下界、三源对照、消融和发布门禁都已有对应实现。
- **论文证据配套度不足：** 当前 clean-v2 发布只有 80 条、全部为 oracle、人工审计数为 0、矛盾正例为 0、测试集仅 16 条；它只能支撑“流水线试验”，不能支撑高水平论文中的基准、鲁棒性或优越性结论。
- **最重要的改进已落地：** 修复了论文主标签在模型加载时丢失的问题；修复了旧布尔配置可能悄悄跑错消融设置的问题；将发布复现实验的审计命令提升为严格门禁。

## 本次已改进

1. **主标签不再丢失。** tier_exceedance_gap_labels 已在 JSON Schema 中定义，却没有进入 ClaimEvidenceSample；数据集经过模型校验后会丢掉它。现已加入模型字段与合法值校验，保证论文主标签、数据集和审计器一致。
2. **消融配置改为“冲突即失败”。** 旧的 shared_threshold=False 可能被默认 calibration_mode 覆盖，造成“名称是无校准，实际跑了共享阈值”的风险。现在兼容旧配置，同时拒绝冲突配置。
3. **参数文档与运行时一致。** 配置中将不存在的 difference 改为真实支持的 severity，并补全 no_calibration 和 weighted_sum。
4. **文稿中的标签定义已对齐。** 将原文“由 tier exceedance 派生 evidence_gap_labels”的表述改为派生 tier_exceedance_gap_labels；evidence_gap_labels 明确为特征级理由。
5. **文稿中的 scope 截断表述已收敛。** 当前代码没有 scope 的硬上限，原附录已改为通过映射和单调性消融评估 scope，避免把计划中的机制写成已实现结果。
6. **发布复现命令更严格。** 生成的审计命令现在必须检查完整三源样本、多人标注和对抗元数据；旧发布包仍是历史试验，不能手工改写成通过。
7. **对照文档已更新。** paper_code_alignment.md 不再把已经完成的功能误标为缺失，并清楚标出默认配置未启用、或当前数据尚未验证的部分。

## 高水平投稿的优先级

1. **先补数据，再加结构。** 收集真实、可审计的三源样本；每个逻辑样本须具备 oracle、extracted、confidence-aware 三条记录，并至少包含一个矛盾正例层。
2. **把论文中的“实现”与“证据”分开。** 当前可讲清模型机制和端到端流程；性能、鲁棒性、图分支和基线优势必须等待完整实验。
3. **对齐配置与叙述。** 若论文重点强调 R(k) 映射或扩展维度，必须实际启用相应配置并做预注册消融；否则它们应降为可选扩展。
4. **重新生成发布包。** 以严格门禁完整跑通训练、开发集阈值拟合、测试、基线、消融、三源对照、对抗评估与发布检查。
